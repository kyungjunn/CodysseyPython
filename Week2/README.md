# 📂 파일 형식 비교: 텍스트 파일 vs 이진 파일

데이터를 저장하고 처리하는 방식에 따라 파일은 크게 **텍스트 파일**과 **이진 파일**로 나뉩니다.  
각 형식의 개념과 장단점을 이해하면 상황에 맞는 최적의 파일 형식을 선택할 수 있습니다.

---

## 1. 개념 정의

### 📝 텍스트 파일 (Text File)
- 데이터를 **문자 인코딩(UTF-8, ASCII 등)** 기반으로 저장합니다.
- 사람이 직접 읽고 편집할 수 있는 형식입니다.
- 대표 확장자: `.txt`, `.csv`, `.md`, `.json`, `.xml`

### 🔢 이진 파일 (Binary File)
- 데이터를 **비트(0과 1)** 단위로, 컴퓨터 메모리 구조 그대로 저장합니다.
- 전용 프로그램이나 라이브러리 없이는 내용을 해석할 수 없습니다.
- 대표 확장자: `.bin`, `.dat`, `.exe`, `.pkl`, `.jpg`

---

## 2. 핵심 차이점 비교

| 구분 | 텍스트 파일 | 이진 파일 |
|:---:|:---:|:---:|
| **저장 방식** | 문자 인코딩 기반 | 비트(0, 1) 단위 |
| **가독성** | ✅ 사람이 직접 읽기 가능 | ❌ 전용 도구 필요 |
| **처리 속도** | 🐢 변환 과정으로 비교적 느림 | ⚡ 파싱 없이 매우 빠름 |
| **파일 용량** | 숫자도 문자로 저장해 상대적으로 큼 | 압축적 저장으로 상대적으로 작음 |
| **보안성** | 낮음 (누구나 열람 가능) | 높음 (즉시 해독 어려움) |
| **호환성** | 높음 (어느 환경에서나 사용 가능) | 낮음 (특정 언어·라이브러리 의존) |

---

## 3. 장단점 상세 분석

### ✅ 텍스트 파일의 장점

- **범용성**: 메모장, VS Code, 엑셀 등 별도 설치 없이 어디서나 열 수 있습니다.
- **유지보수 용이**: 데이터 오류 발생 시 사람이 직접 눈으로 확인하고 수정할 수 있습니다.
- **버전 관리 최적화**: Git 등의 도구에서 **행(line) 단위**로 변경 사항을 추적하기에 이상적입니다.
- **디버깅 편의**: 출력 내용을 즉시 육안으로 확인할 수 있어 개발 중 오류를 찾기 쉽습니다.

### ❌ 텍스트 파일의 단점

- **저장 공간 낭비 가능**: 정수 `12345`를 문자 5개로 저장하므로, 숫자 데이터가 많을수록 비효율적입니다.
- **보안 취약**: 누구나 파일을 열어 내용을 쉽게 확인하거나 무단으로 수정할 수 있습니다.
- **처리 속도**: 데이터를 읽을 때 문자 → 원시 타입 변환(파싱) 과정이 필요해 속도가 느립니다.

---

### ✅ 이진 파일의 장점

- **고속 처리**: 파싱 과정 없이 메모리에 직접 로드되므로 **대용량 데이터 처리 속도**가 압도적으로 빠릅니다.
- **데이터 정밀도 보존**: 리스트, 딕셔너리 등 복잡한 객체 구조를 손실 없이 그대로 저장하고 복원할 수 있습니다.
- **1차 보안 효과**: 일반적인 방법으로는 내용을 즉시 해독하기 어려워 기초적인 보안 역할을 합니다.
- **저장 효율**: 데이터 타입 그대로 저장하므로 텍스트 파일 대비 용량이 작을 수 있습니다.

### ❌ 이진 파일의 단점

- **가독성 없음**: 일반 에디터에서 열면 깨진 문자(`^@`, `��` 등)만 표시되어 내용 파악이 불가능합니다.
- **플랫폼·라이브러리 의존**: Python의 `pickle`, Java의 `Serializable` 등 특정 환경에 종속되는 경우가 많습니다.
- **버전 관리 어려움**: Git에서 바이너리 파일은 변경 내용을 줄 단위로 추적할 수 없어 diff가 불가능합니다.
- **디버깅 난이도**: 데이터가 손상되어도 어느 부분이 잘못되었는지 파악하기 매우 어렵습니다.

## 4. 언제 어떤 형식을 선택할까?
```text
📋 텍스트 파일을 선택해야 할 때
  - 사람이 직접 읽고 수정해야 하는 설정 파일, 로그, 문서
  - Git으로 버전을 관리해야 하는 소스코드, 데이터셋
  - 다양한 시스템 간 호환성이 중요한 데이터 교환

⚙️ 이진 파일을 선택해야 할 때
  - 대용량 데이터를 빠르게 읽고 써야 하는 경우
  - 복잡한 객체나 모델(AI 가중치 등)을 그대로 저장할 때
  - 처리 속도와 저장 효율이 최우선인 내부 캐시·시스템 데이터
```

## 5. 실제 활용 예시
```python
# 텍스트 파일 쓰기 / 읽기
with open("data.csv", "w", encoding="utf-8") as f:
    f.write("name,flammable\nHydrogen,True\n")

# 이진 파일 쓰기 / 읽기 (Python pickle)
import pickle

data = {"name": "Hydrogen", "flammable": True}

with open("data.bin", "wb") as f:
    pickle.dump(data, f)

with open("data.bin", "rb") as f:
    loaded = pickle.load(f)
```

---

> 💡 **핵심 원칙**: 사람이 자주 확인하고 수정해야 하는 데이터는 **텍스트 파일**,  
> 시스템 내부에서 빠르게 처리해야 하는 데이터는 **이진 파일**을 사용하세요.
