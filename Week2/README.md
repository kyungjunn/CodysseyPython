# ------------------------------------------------------------

# 1\. 데이터 비교 요약 (Comparison Table)

# ------------------------------------------------------------

# 

# \[ 구분 ]       | \[ 텍스트 파일 (Text) ]   | \[ 이진 파일 (Binary) ]

# --------------+-------------------------+-------------------------

# 파일 확장자   | .csv, .txt, .md         | .bin, .dat, .pickle

# 데이터 구조   | 문자 인코딩(UTF-8) 기반 | 비트(0, 1) 단위 데이터

# 가독성        | 사람이 읽고 수정 가능   | 전용 프로그램 필수

# 처리 속도     | 변환 과정 필요 (느림)   | 메모리 구조 사용 (빠름)

# 

# ------------------------------------------------------------

# 2\. 상세 장단점 비교

# ------------------------------------------------------------

# 

# 📝 텍스트 파일 (Text File)

# &nbsp;  - 예: Mars\_Base\_Inventory\_List.csv

# 

# &nbsp;  (+) 장점:

# &nbsp;      \* 범용성: 메모장, 엑셀, VS Code 등 어디서나 열람 가능

# &nbsp;      \* 유지보수: 데이터 오류 발생 시 사람이 즉시 수정 가능

# &nbsp;      \* 버전 관리: Git 등을 통한 행 단위 변경 추적 최적화

# 

# &nbsp;  (-) 단점:

# &nbsp;      \* 용량: 숫자도 문자로 저장하여 저장 공간 낭비 발생

# &nbsp;      \* 보안: 누구나 내용을 쉽게 열람할 수 있어 보안에 취약

# 

# 🔢 이진 파일 (Binary File)

# &nbsp;  - 예: Mars\_Base\_Inventory\_List.bin

# 

# &nbsp;  (+) 장점:

# &nbsp;      \* 효율성: 파싱 과정이 없어 대용량 처리 시 압도적 속도

# &nbsp;      \* 정밀도: 복잡한 객체 구조를 깨짐 없이 그대로 보존

# &nbsp;      \* 보안성: 일반 에디터로는 내용을 알 수 없는 보안 효과

# 

# &nbsp;  (-) 단점:

# &nbsp;      \* 가독성: 에디터에서 열 경우 인코딩 오류(외계어) 발생

# &nbsp;      \* 호환성: 특정 언어나 라이브러리에 의존적인 경우 많음

# 

# ------------------------------------------------------------

# 3\. 결론 및 적용 (Application)

# ------------------------------------------------------------

# 

# \* 상시 확인용: 목록 데이터는 \[CSV] 형식으로 관리

# \* 시스템 내부: 빠른 전송 및 내부 캐시는 \[BIN] 형식 사용

# => 운영의 안정성과 처리의 효율성을 동시에 확보함.
